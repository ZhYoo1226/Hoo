基于上述对话历史，判断最近的用户消息是否引入了一个**全新的、尚未分配任务ID**的任务。根据判断结果输出一个JSON决策。

## 判断规则
1. **全新任务 (meta:task)**
   - 最近的用户消息表达了一个新的、独立的任务意图（查询、分析、探索等）。
   - 对话历史中**没有**出现格式为 `<执行>: 创建任务->... (task_id:...)` 的记录，或者历史虽有其他任务，但当前消息明确开始了一个不同主题的新任务。
   - 此时你必须创建新任务，输出 `action: "meta:task"`，并完整填写 `name`、`type`、`task`、`goal`、`steps` 字段。

2. **已有任务继续 (predict:continue)**
   - 最近的用户消息是对一个**已分配任务ID**的任务的延续、追问、确认、修改或无关闲聊（如“谢谢”）。
   - 判断依据：历史对话中**已存在** `观察: 创建任务->... (task_id:...)`，且当前消息未脱离该任务的主线。
   - 此时你输出 `action: "predict:continue"`，**无需**输出其他字段。

3. 任务ID以系统观察消息中的 `(task_id:数字)` 为准，一旦出现即视为任务已创建。

## 输出格式
**你必须以 `### 分析创建任务` 开头**，然后输出一个JSON对象，并用 ```action 代码块包裹。

**当 action 为 meta:task 时的完整结构：**
```action
{
    "action": "meta:task",
    "name": "任务名称",
    "type": "查询统计 | 业务探索 | 搜索问答",
    "task": "任务的具体需求描述",
    "goal": "任务最终要达成的目标",
    "steps": ["规划的步骤1", "步骤2", ...]
}
```

**当 action 为 predict:continue 时，只输出：**
```action
{
    "action": "predict:continue"
}
```

## 示例

### 示例1：已有任务，用户说“谢谢”
**历史对话记录**
```
用户: 帮我查一下上个月华东区的销售额。
观察: 创建任务->查一下上月华东区的销售额(task_id:20260518010101)
助手: 好的，正在为您查询，请稍等……
（查询结果略）
用户: 谢谢。
```
**分析**
最近消息“谢谢”是结束语，任务ID已存在，无需新建任务。
### 分析创建任务
```action
{
  "action": "predict:continue"
}
```

### 示例2：全新任务，首次提出需求
**历史对话记录**
```
用户: 你好，请分析一下今年第二季度各产品线的利润变化趋势。
```
**分析**
对话无任何任务ID记录，这是一个全新的业务探索任务。
### 分析创建任务
```action
{
  "action": "meta:task",
  "name": "第二季度产品线利润趋势分析",
  "type": "统计分析",
  "task": "分析今年第二季度各产品线的利润变化趋势",
  "goal": "生成各产品线利润的同比/环比变化报告，找出关键波动因素",
  "steps": [
    "提取第二季度各产品线的收入与成本数据",
    "计算各产品线的利润及同比增长率",
    "绘制趋势图表并识别异常波动",
    "汇总分析结论"
  ]
}
```

### 示例3：已有任务，用户追加细化条件
**历史对话记录**
```
用户: 统计本月新增客户数。
观察: 创建任务->统计本月新增客户数(task_id:20260518020202)
助手: 正在统计……
用户: 能按省份分一下吗？
```
**分析**
“按省份分一下”是对任务ID 20260518020202 的细化追问，属于继续当前任务。
### 分析创建任务
```action
{
  "action": "predict:continue"
}
```
