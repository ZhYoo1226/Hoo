基于对话历史，判断最近用户消息是否提出了一个与已有任务不同的全新任务。若是，提取任务信息并以 JSON 输出；若不是（如追问、细化、纠错或与已有任务重复），只输出“无”。

## 输出格式
你必须以 `### 分析创建任务` 作为单独一行开头，然后紧跟一个 `action` 代码块，其中包含 JSON 对象。
```action
{
    "action": "meta:task",
    "name": "任务名称",
    "type": "查询统计 | 业务探索 | 搜索问答",
    "task": "任务的具体需求描述",
    "goal": "任务最终要达成的目标"
}
```

## 决策规则
- 历史中已有旧任务，但当前消息**明确切换了主题、领域或目标**，与所有历史任务无关（不是追问、细化或纠错）。
- 新任务与 `## 已有任务列表` 中的任务**不能**重复、重叠或冲突。若高度相似，应视为继续旧任务。
