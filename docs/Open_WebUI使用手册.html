Open WebUI 使用手册

OpenSolver 的 Web 对话前端 — 通过浏览器与智能体交互

什么是 Open WebUI

Open WebUI 是一个开源的 AI 对话前端，提供类似 ChatGPT 的 Web 界面。它通过 OpenAI 兼容 API 连接到 OpenSolver Gateway，让用户可以通过浏览器与智能体对话。

启动方式

OpenSolver 启动时会自动拉起 Gateway 和 Open WebUI，无需单独操作。

一键启动

PS> .venv312\Scripts\activate
PS> python app.py

启动后访问：

  • Open WebUI：http://localhost:3000
  • Gateway API：http://localhost:8081

配置开关

在 config.yaml 中控制各组件的启停：

配置项默认值说明
gateway.enabledtrue是否启动 OpenAI 兼容 API 网关（端口 8081）
gateway.port8081网关监听端口
webui.enabledtrue是否自动启动 Open WebUI（端口 3000）
webui.port3000WebUI 监听端口

环境变量（OPENAI_API_BASE_URL 等）由 app.py 自动设置，无需手动配置。

首次使用：创建管理员账号

  1. 填写名称、电子邮箱、密码
  2. 点击创建管理员账号
  3. 之后用该账号登录

账号数据存储在 d:\solver\.webui_secret_key 和 SQLite 数据库中（open-webui 包自动管理）。

基本功能

新建对话

登录后点击左上角新对话，选择模型（自动识别 OpenSolver 配置的模型），即可开始聊天。

消息流转路径：

浏览器 → Open WebUI → POST /v1/chat/completions → OpenSolver Gateway → FSM ThinkClient → LLM(qwen3) → 响应 → SSE 流式返回 → 浏览器

对话历史

Open WebUI 自动保存所有对话，左侧栏可查看和管理历史会话。

注意：Open WebUI 的对话历史与 OpenSolver 的聊天记录（workspace/chat/）是两套独立系统，互不影响。

模型管理

Open WebUI 自动从 /v1/models 拉取可用模型。你可以在设置中添加多个 OpenAI 兼容后端，从而在一个界面切换不同模型。

常见问题

网页打不开

检查 OpenSolver 是否正常运行：

PS> curl http://localhost:8081/health

发消息没回复

查看 OpenSolver 终端日志，确认 ThinkClient 线程是否正在处理。检查 LLM API 服务（183.222.234.174:5090）是否可达。

RAG 知识库报错

RAG_EMBEDDING_ENGINE=openai 时，上传文档做知识库检索会调用 /v1/embeddings 接口。OpenSolver 当前未实现该端点，知识库功能暂时不可用。基础聊天不受影响。

如果报错影响使用，可以去掉该变量让 embedding 引擎为空（知识库功能仍不可用，但不会尝试调用不存在的接口）。

停止服务

在 OpenSolver 终端按 Ctrl+C 即可。OpenSolver 会优雅退出，自动停止 Gateway 和 Open WebUI（执行 stop_gateway()、stop_webui() 和 g_owner.destroy()）。
